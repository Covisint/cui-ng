Password-Validation
Password-validation is a directive made to easily display password syntax rules and validation. This is made to work with covisint API password rules - should you want to set custom password rules use custom-error instead.
Using this password together with ng-messages will easily allow you to create a validated password input, such as this one.

Passwords must:

not have any lowercase letters
not have any upper case letters
not have any numbers
not have any special symbols (ex: ! * + )
not contain any of the following chars: ^&*)(#$
have between 8-18 characters

have 3 of the following:
at least one lower case letter
at least one upper case letter
at least one number
at least 1 special symbol
(ex: ! * + )
